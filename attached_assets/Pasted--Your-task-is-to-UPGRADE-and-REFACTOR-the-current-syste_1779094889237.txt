==================================================
FIELD RULES
===========

TITLE (logic)

* Clean HTML entities
* Trim whitespace
* Capitalize first letter
* Max 120 chars

Priority:

1. CSV title
2. Manual override
3. Sunsky cleanup

Fallback:
original Sunsky title

---

SLUG (derive)

* Transliterate unicode
* Lowercase
* Replace spaces with "-"
* Remove special chars
* Max 70 chars
* Ensure DB uniqueness
* On duplicate:
  append -{sku[-4:]}

Fallback:
product-{sku[-8:]}

---

TAGS (logic)

* Exactly 3 tags:

  * primary category
  * subcategory
  * product type
* Normalize casing
* No brand names
* No generic tags

Fallback:
primary category

---

IMAGE ALT TEXT (derive)
Format:
{cleaned_title} – {primary_attribute} – {brand}

Rules:

* Max 125 chars
* Trim last word safely

Fallback:
{cleaned_title} – {sku}

---

IMAGE FILE NAMES (derive)
Format:
{slug}-{index}.webp

Rules:

* Sequential indexing
* Max 70 chars

---

DESCRIPTION (ai)
Input:

* product name
* category
* paramsTable attributes
* top keywords

Rules:

* 150–250 words
* No HTML
* Structure:

  * intro
  * features
  * benefits
  * compatibility
  * closing

Banned phrases:

* "the best"
* "100%"
* "guarantee"

Retry:
3 times with exponential backoff

Fallback:
cleaned Sunsky description

---

SHORT DESCRIPTION (derive from Description)
Rules:

* 2–3 sentences
* Max 400 chars
* First sentence contains primary keyword
* SEO-oriented

Fallback:
first sentence from Description

---

META TITLE (derive)
Format:
{cleaned_title} | {brand}

Rules:

* Max 60 chars
* Trim title safely

Fallback:
cleaned_title only

---

META DESCRIPTION (derive from Description)
Rules:

* 140–160 chars
* Include primary keyword
* Natural CTA
* SEO-oriented
* No keyword stuffing

Fallback:
trim Description to 155 chars
