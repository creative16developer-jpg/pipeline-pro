System Overview

The system automates importing products from the Sunsky supplier catalogue into your WooCommerce store. For every product it:

  • Downloads and processes images (resize, watermark, convert to WebP)
  • Uses AI to extract product attributes (color, size, material, etc.)
  • Pauses for your review before uploading anything
  • Uploads products as drafts — nothing goes public until you publish
Sunsky Catalogue → Pipeline → Your Review → WooCommerce (drafts) ───────────────────────────────────────────────────────── Settings you configure once: WooCommerce Attributes (synced from your store) ↓ AI Extraction Rules (confidence thresholds per attribute) ↓ Attribute Profiles (groups of attributes per product type) ↓ Category Mappings (Sunsky category → WooCommerce + Profile) ───────────────────────────────────────────────────────── Each pipeline run: Fetch products → Process images → AI extracts attrs ↓ ⏸ You review & confirm ↓ Upload as WooCommerce drafts
Nothing uploads without your approval. The pipeline always pauses at the Attribute Review step. You see exactly what the AI extracted — with confidence scores — before clicking Confirm.
1

Sync WooCommerce Attributes

Settings → Attribute Profiles

Before anything else, the system needs to know what attributes exist in your WooCommerce store. Click Sync Attributes to pull them in.

This is a one-time setup, but you can re-sync any time you add new attributes in WooCommerce.

Example — Attributes synced from your store
Attribute NameAvailable Terms (Values)
colorRed, Blue, Black, White, Navy, Green, Yellow…
sizeXS, S, M, L, XL, XXL, One Size
materialCotton, Polyester, Nylon, Wool, Linen…
genderMen, Women, Unisex
Important: Attributes must be created in WooCommerce first (via your WooCommerce admin panel). The system reads from WooCommerce — it does not create attributes for you.
2

Set AI Extraction Rules

Settings → AI Extraction Rules

For each attribute, you can optionally set a confidence threshold — how certain the AI must be before it auto-approves a value. If the AI is below that threshold, it still extracts the value but flags it for your manual confirmation.

Example — Extraction rules for a clothing store
AttributeConfidence ThresholdReasoning
color 75% Colors are usually clear in product descriptions
size 90% Sizes are critical — wrong size means returns
material 60% Material can be vague in supplier data; lower bar is fine
gender 80% Usually clear from category name or title

If no rule is set for an attribute, a default threshold of 70% is used automatically.

How confidence works in practice: If the AI sees "100% Cotton" in a product description and extracts material = Cotton with 94% confidence — and your threshold is 60% — it auto-approves. If it's only 55% confident, it shows the value to you and asks you to confirm or override.
3

Create Attribute Profiles

Settings → Attribute Profiles

An Attribute Profile is a named list of attributes that a type of product needs. You create one profile per product category type, then reuse it across many category mappings.

Example — Three profiles for different product types
Profile NameAttributes IncludedUsed For
Clothing color, size, material, gender T-shirts, dresses, jackets, trousers…
Electronics connectivity, voltage, battery_life, weight Chargers, earphones, cables, gadgets…
Accessories color, material, dimensions Bags, wallets, belts, hats…

Only attributes that exist in your WooCommerce store (synced in Step 1) can be added to a profile. This ensures the AI only extracts values that WooCommerce can actually use.

4

Map Categories

Settings → Category Mapping

Category Mapping is where you connect everything together. For each Sunsky supplier category, you tell the system:

  • Which WooCommerce category the products should go into
  • Which Attribute Profile to use (so the AI knows what to extract)
Example — Category mappings for a clothing & electronics store
Sunsky CategoryWooCommerce CategoryAttribute Profile
Men's T-Shirts Clothing > Men > Tops Clothing
Women's Dresses Clothing > Women > Dresses Clothing
Winter Jackets Clothing > Outerwear Clothing
USB Chargers Electronics > Charging Electronics
Bluetooth Earphones Electronics > Audio Electronics
Handbags Accessories > Bags Accessories
One-time setup: You create these mappings once. Every time you run a pipeline for "Men's T-Shirts", the system automatically knows to use the Clothing profile and the Clothing > Men > Tops WooCommerce category.
5

Run a Pipeline

Pipelines → + New Pipeline

When you want to import products, you start a Pipeline. Select your store and category — the system does the rest automatically.

📥

Fetch products from Sunsky

Downloads all products in the selected category — titles, descriptions, specifications, images.

🖼️

Process images

Each product image is resized, watermarked with your branding, and converted to WebP format for fast loading.

🤖

AI extracts attributes

The AI reads each product's data and extracts the attributes defined in the profile — e.g. color, size, material — with a confidence score for each.

⏸

Pause — Attribute Review

You review all extracted values. The pipeline waits here until you click Confirm & continue. See Step 6 for full details.

☁️

Upload to WooCommerce as drafts

Products are uploaded with all attributes, images, and categories — but set to Draft status so nothing is publicly visible yet.

✅

Publish in WooCommerce

You do a final review in WooCommerce and publish the products you are happy with. This step is done directly in your WooCommerce admin.

6

Attribute Review — The Key Step

Pipelines → click running pipeline

This is the most important step. After the AI has processed every product, the pipeline pauses and shows you a detailed review panel. You see every product with every attribute the AI extracted — including the value and how confident the AI was.

Example — Review panel for "Classic Men's Polo Shirt" (SKU: SMS-4421)
Attribute AI Extracted Value Confidence Status
color Navy Blue 94% ✓ Auto-approved
size M 88% ✓ Auto-approved
material 100% Cotton 72% ⚠ Needs confirm
gender (not found) — ✗ Unset — pick value

For each attribute, one of three things happens:

StatusWhat it meansWhat you do
✓ Auto-approved AI was confident enough (above your threshold). Value is shown so you can verify it. Nothing — just read it. If it looks wrong, you can still override it.
⚠ Needs confirm AI extracted a value but was below your confidence threshold. It could be right — it just wants your sign-off. Click Confirm to accept the AI value, or pick a different value from the dropdown.
✗ Unset AI could not extract a value for this attribute from the product data. Pick the correct value from the WooCommerce terms dropdown.

Filter tabs let you focus: All Products · Needs Review (only products with amber/red attributes) · Auto-resolved (products the AI handled fully).

Once all flagged attributes are handled, the Confirm & continue › button becomes active. Clicking it saves your decisions and the pipeline uploads the products to WooCommerce.

The system gets smarter over time

Every time you correct an AI value, that correction is saved as a normalisation rule. For example, if the AI extracted "COTTON 100%" and you corrected it to "Cotton", the system remembers that mapping and applies it automatically to every future product — so you never have to correct that same thing twice.

Key Points Summary

🔒

Nothing uploads without your approval

The pipeline always pauses at Attribute Review. You see every value the AI extracted before anything is sent to WooCommerce.

📝

Products upload as drafts

All imported products land in WooCommerce as Drafts. Nothing is publicly visible until you manually publish them in WooCommerce.

🧠

AI learns from your corrections

Each confirmed or corrected value teaches the system. Future runs for the same category will need less and less manual review.

♻️

Set up once, reuse forever

Create your profiles and category mappings once. Every future pipeline for "Men's T-Shirts" automatically uses the right settings.

🎚️

You control confidence thresholds

Set stricter thresholds for critical attributes like size, and looser ones for less important fields. The AI auto-approves everything above your bar.

🖼️

Images processed automatically

Every image is resized, watermarked with your branding, and converted to WebP — no manual image editing needed.

Glossary

TermDefinition
WooCommerce Attribute A product property in WooCommerce (e.g., color, size). Attributes have a fixed list of valid values called terms.
AI Extraction Rule A per-attribute setting that tells the AI how confident it must be before auto-approving a value. Below the threshold, the value is flagged for your review.
Attribute Profile A named collection of attributes expected for a type of product (e.g., a "Clothing" profile includes color, size, material). Assigned to a category mapping.
Category Mapping A rule that links a Sunsky supplier category to a WooCommerce category, along with the Attribute Profile to use for that category's products.
Pipeline A single import run for one store + category combination. Covers fetch → images → AI → review → upload.
Confidence Score A percentage (0–100%) indicating how certain the AI is about an extracted value. Higher = more certain.
Normalisation Rule A saved correction that maps a raw supplier value to the correct WooCommerce term (e.g., "COTTON 100%" → "Cotton"). Applied automatically on future runs.
Draft A WooCommerce product status meaning the product exists in your store but is not publicly visible. You publish it manually when ready.
Sunsky The supplier API that provides the product catalogue, descriptions, specifications, and images being imported.
